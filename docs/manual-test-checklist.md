# Manual acceptance checklist

## Import and triage

- Import an old CSV/XLSX case and confirm it appears in Triage, not active attention queues.
- Close and reopen the wizard after each step; confirm saved values resume.
- Enter historical filing/service dates and confirm no event is stamped as newly completed today.
- Activate and confirm only applicable future alerts are generated.
- Confirm a pre-existing active case is unchanged.

## Dashboard and navigation

- Confirm the default dashboard contains active cases only and the triage count opens the triage list.
- Mark/clear Waiting from a quick action and verify holder/reason plus activity.
- Select several cases, bulk defer to one date, confirm the summary, and verify an activity entry per case.
- Clear a deferral and confirm queue placement recalculates.
- Open filters, set several filters, close/reopen, verify active count and clear-all.
- Search by case/party/case number/job/tract; test arrows, Enter, Escape, and click.
- From every case tab, click Cases and confirm the main case list opens.

## Case and audit

- Verify header identity, stage, lifecycle, warning, trial date, and wrapping issue tags.
- Add/remove a tag in Edit Case and confirm activity.
- Change stage/status/holder/waiting/deferment and confirm Recent Activity.
- Correct an activity entry with a required reason; confirm Edited and original history.

## Service and publication

- Change service status and confirm the UI says updated, not “perfected today.”
- Enter an actual service date separately.
- Save first and second publication dates independently.
- Confirm second-before-first is rejected and newspaper is requested when a date exists.
- Confirm legacy service/publication notes are read-only and no Edit Case notes control exists.

## Tasks, deadlines, and discovery

- Advance a stage and exercise Generate All, Review, and Do Not Generate.
- Add task/deadline from a filtered template; verify likely duplicates are warned/skipped.
- Override a calculated deadline and verify old/new/reason/timestamp history.
- Confirm only one discovery type is expanded initially and summary counts match.
- Generate with and without issue tags; confirm tag content and no repeated questions.
- Preview/edit before saving; confirm the export and activity identify source tags.
- Change a template and confirm an existing generated document is unchanged.

## Regression

- Verify job and tract remain separate.
- Exercise the 20% fee-shift calculation against known fixtures.
- Create more than 20 backups and confirm only 20 remain.
- Confirm completed tasks/deadlines and generated documents remain accessible.
